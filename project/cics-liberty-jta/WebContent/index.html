Using a Liberty JVM server to co-ordinate transactions

This sample backs the blog post on CICSdev

This sample demonstrates the use of the Java Transaction API (JTA) inside a CICS WebSphere Liberty profile JVM server to co-ordinate transactions involving CICS and a database.

Use the below form to store data to CICS and Derby. The rollback option can be used to force JTA to rollback the transaction after the writes have been performed to CICS and the database, but before those writes are committed.

Note: Derby is purely used here for simplicity. Any database which provides a Java EE DataSource can be used with minimal code updates.
